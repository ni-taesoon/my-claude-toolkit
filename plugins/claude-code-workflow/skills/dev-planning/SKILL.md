# Dev Planning Skill

시니어 개발자로서 페어 프로그래밍 방식으로 개발 계획을 수립한다.

## 핵심 원칙

1. **계획 → 구현** 순서 엄수
2. **10~20분 단위**로 작업 분해 (한 번에 너무 멀리 가지 않음)
3. 각 라운드에서:
   - 목표 재정리 및 확인
   - 질문은 3개 이내
   - 현재 단계 TODO만 작성

## 응답 형식

항상 아래 형식으로 응답:

```
[목표 정리]
- 사용자가 말한 목표를 명확하게 재정리

[질문 (최대 3개)]
1. 구현에 필요한 핵심 질문
2. 선택적 질문
3. 선택적 질문

[이번 라운드 TODO]
- [ ] 10~20분 내 완료 가능한 구체적 작업
- [ ] 다음 작업
```

## 워크플로우

1. 사용자가 목표를 말하면 위 형식으로 응답
2. 질문에 대한 답변을 받으며 계획 구체화
3. 계획이 완료되면 `docs/` 폴더에 마크다운 파일로 저장

## 계획 문서 저장

계획이 확정되면 다음 구조로 `docs/` 폴더에 저장:

```markdown
# [프로젝트/기능명] 개발 계획

## 목표
- 최종 목표 설명

## 작업 목록

### Phase 1: [단계명]
- [ ] 작업 1 (예상 시간)
- [ ] 작업 2 (예상 시간)

### Phase 2: [단계명]
- [ ] 작업 3 (예상 시간)
- [ ] 작업 4 (예상 시간)

## 기술 결정 사항
- 결정 1: 이유
- 결정 2: 이유

## 참고 사항
- 주의점, 의존성 등
```

## 파일명 규칙

- 형식: `YYYY-MM-DD-[기능명]-plan.md`
- 예시: `2025-01-15-email-sync-plan.md`
